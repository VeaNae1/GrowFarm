# Server
GrowFarm 서버
## Key Features
* 앱에서 전송한 사진 모델로 전달
* 모델에서 나온 결과를 DB에 저장
* 앱에서 요청시 DB의 결과를 반환
## Directions
* ngrok토큰을 받아 서버 코드의 ngrok부분에 붙여넣기
* 서버 실행전 MySQL DB 생성
  > ID, Password, 테이블 등 정보는 서버 코드 내부에 있음
* MySQL DB의 recommendation_info 테이블에 다음 정보 넣기(예시)
  > 1	N	["질소비료(N)", "복합비료(10-10-10)"]	["주기적 토양검사", "유기물 보충"]	["잎이 누렇게 변함", "생장이 늦어짐"]	   
	> 2	P	["인산비료(P₂O₅)", "퇴비 혼합비료"]	["토양 산성화 방지", "유기질 비료 보충"]	["잎이 보라색으로 변함", "뿌리 발달 저해"]	   
	> 3	K	["칼륨비료(KCl)", "복합비료(15-15-15)"]	["수분 조절 강화", "잎 끝 마름 방지"]	["잎 끝이 마르고 갈색 반점 발생", "광합성 저하"]
* 서버 코드 실행 후 나온 서버 주소를 앱의 ApiConfig.kt안에 넣어주기
* 같은 폴더 속에 모델 파일 lack_classify.pt, leaf_detect.pt 존재해야 함
